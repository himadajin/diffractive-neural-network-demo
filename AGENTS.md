# AGENTS.md

## Project

Interactive visual artwork themed around optical diffraction and neural networks.
It is not a physically rigorous optical simulator; visual plausibility and the
artwork's design language matter most.

Read `docs/CONTEXT.md` for project vocabulary before changing visual behavior.

## Commands

- Install dependencies: `npm install`
- Start dev server: `npm run dev`
- Type-check: `npm run check`
- Build: `npm run build`
- Format: `npm run format`

There is no lint or test command yet. Do not invent one.

## Before Commit

- Run `npm run check` for code changes.
- Run `npm run build` when runtime behavior, static assets, dependencies, or
  build configuration change.
- Run `npm run format` after broad edits or when formatting changed files.

## Formatting

- Use the repository Prettier configuration in `.prettierrc.json`.
- Do not format generated or binary assets ignored by `.prettierignore`, including
  `public/ort/`, `public/models/`, and `dist/`.
- Do not introduce a separate formatter or competing style configuration unless
  the project explicitly adopts a new tool.

## Visual QA

For UI, Three.js, Canvas, camera, classifier, or visual-behavior changes, verify
the app in a browser. Check at least:

- idle state before drawing
- drawn input and Output Screen response
- `?debug=1` when camera or debug controls change

Also check browser console warnings/errors and summarize the visual result.

## Assets

- `public/models/mnist-8.onnx` is committed and documented in
  `THIRD_PARTY_NOTICES.md`.
- `public/ort/` is generated by `npm run prepare:ort` from `onnxruntime-web` and
  must not be committed.
